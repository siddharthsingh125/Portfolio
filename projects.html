A/B Testing | E-Commerce Conversion

Cart Page Upsell Module Optimization

A Conversion Rate Optimization case study delivering Average Order Value growth.

Challenge

The cart page offered no opportunity to promote complementary products before checkout, limiting the potential to increase Average Order Value (AOV). The goal was to introduce relevant upsell options without disrupting the customer's purchase journey or creating unnecessary friction.

Solution

We designed a lightweight upsell module featuring two optional accessories—Chain Extender and Gift Box. Customers could add either item with a single click directly from the cart page, while those who chose not to engage experienced no changes to their cart, ensuring a seamless checkout flow.

Our Approach

We began by reviewing the client's hypothesis and understanding the desired business outcome. After analyzing the existing cart experience, we identified the ideal placement for the upsell component and validated product eligibility with the client. The solution was implemented using a clean, non-intrusive interface that aligned with the existing design system. Finally, comprehensive Manual QA and Cross-Browser Testing ensured reliable functionality before deployment.

Key Features & Implementation

  • Introduced two optional cart-page upsell products.
  • One-click opt-in functionality for a frictionless experience.
  • Automatic cart update when an upsell is selected.
  • Cart total remains unchanged if no option is selected.
  • Responsive implementation across desktop, tablet, and mobile.
  • Cross-browser compatibility testing.
  • Manual QA testing to validate cart behavior.
  • Maintained consistent UI with the existing checkout experience.

Impact Delivered

The implementation provided a scalable upsell framework that supports future CRO initiatives while preserving a smooth checkout experience. The experiment was designed to encourage higher Average Order Value, improve upsell adoption, and generate measurable insights through A/B Testing without introducing unnecessary user friction.

A/B Testing | E-Commerce / Health & Wellness

Sliding Mini Cart Redesign & Checkout

Interactive sliding cart development bypassing API limitations to boost checkout flow.

Challenge

The existing notification-style mini cart had several technical limitations that made redesigning it impractical. It displayed only one product at a time, lacked quantity controls, and exposed limited product information, making it difficult to implement the new CRO-focused design while preserving existing functionality.

Solution

Instead of heavily modifying the existing component, a completely new sliding mini cart was developed. Product data was fetched using the existing cart APIs and rendered inside the new interface, allowing multiple cart items, quantity management, product removal, dynamic pricing, and a modern checkout experience without disrupting the backend cart logic.

Our Approach

After reviewing the existing cart architecture, we identified that extending the original mini cart would introduce unnecessary complexity due to its DOM structure and limited functionality. We collaborated with the client to confirm expected behavior, then built a reusable sliding cart component that consumed the existing cart API. The new component maintained synchronization with the backend while providing a significantly improved user experience through responsive interactions and optimized UI.

Key Features & Implementation

  • Built a custom sliding mini cart from scratch.
  • Integrated existing cart APIs to fetch live cart data dynamically.
  • Displayed multiple products simultaneously.
  • Implemented quantity increase and decrease functionality.
  • Added product removal capability directly in the drawer.
  • Supported dynamic pricing including discounted products.
  • Displayed sale and original prices side-by-side when applicable.
  • Added a prominent Checkout Call to Action.
  • Responsive layout optimized for desktop and mobile viewports.
  • Smooth open and close cart drawer animations.
  • Real-time cart updates without full page reloads.
  • Performed comprehensive Manual QA and Cross-Browser Testing.

Impact Delivered

The redesigned sliding cart created a more intuitive shopping experience by giving customers complete visibility into their cart without interrupting their browsing journey. The implementation reduced interaction friction, simplified cart management, and provided a scalable frontend architecture suitable for future CRO experiments and feature enhancements.
